FAQ

How does this analyzer work?

This analyzer creates a graph database while java compiles your project using neo4j. After this, it runs a set of user defined queries that search for common programming mispractices that are not detected by the java compiler.

How do I create a query?

The queries are written in cipher, which is the language that neo4j uses for its queries. The data model is not available yet, since it is not the final one.

This filed must be returned:

  • column
  • line
  • file: it is in the enclusingCU "fileName" property.

This is neccessary in order for it to work properly, for instance:

RETURN retExpr.lineNumber as line, retExpr.column as column , enclosingCU.fileName as file

Also, this match must be included:

MATCH (enclosingCU)<-[:PACKAGE_HAS_COMPILATION_UNIT]-()<-[:PROGRAM_DECLARES_PACKAGE]-(program) WHERE program.ID=$programID

This way the query only searches one program, not all of them.

How do I fix an error?

The definition of the query should give enough info to fix the problem.

Why I can't name my query?

If you have problems adding a query because the name is not correct, this must be because you are not following the naming standarts.
