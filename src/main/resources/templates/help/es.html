FAQ

¿Cómo funciona este analizador?

Este analizador crea una base de datos de grafo mientras java compila tu proyecto usando neo4j. Después, ejecuta una serie de queries definidas por el usuario que buscan por malas prácticas de programación que no son detectadas por el compilador.

¿Como creo una consulta?

Las consultas estan escritas en cipher, que es el lenguage que neo4j usa para las consultas. El modelo de datos que genera no es definitivo, por lo tanto este documento aún no lo incluye. En el futuro habrá un enlace al modelo para que se puedan diseñar consultas más facilmente.

Para que las queries detecten errores correctamente tiene que seguir unos criterios contretos. Las únicas condiciones es que devuelva los siguientes campos:

  • Column: la columna donde se encuentra el error
  • Line: la linea donde se encuantra el error
  • File: esta propiedad esta en "fileName" en enclosingCU

Esto es necesario para que funcione. Por ejemplo:

RETURN retExpr.lineNumber as line, retExpr.column as column , enclosingCU.fileName as file

Y además, este match tiene que estar presente:

MATCH (enclosingCU)<-[:PACKAGE_HAS_COMPILATION_UNIT]-()<-[:PROGRAM_DECLARES_PACKAGE]-(program) WHERE program.ID=$programID

De esta manera los errores son buscados dentro del programa actual. Si esto se omite, va a buscar en TODOS los programas en la base de datos.

¿Como arreglo un error detectado en mi código?

La definición del error debería ser suficiente para arreglarlo. Si no lo es, eso es problema del creador de la consultar original.

¿Porque no puedo crear una consulta con el nombre que quiero?

Si tienes problemas al crear una query es porque seguramente las estés nombrando incorrectamente.

Su convenio de nombrado es como los paquetes de java. Por ejemplo "es.uniovi.myQuery" es válido, mientras que "es.test-1" o "es..test" no lo son.
